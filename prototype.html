I don't know where to start with this problem. Can you help?
Mentor

Great question! Let's think about this step by step.

Key Insight

The brute force approach would check every pair, giving us O(n²) time. But we can do better.

Think about it: for each number, what specific value would you need to find to complete the sum?

What data structure lets you check "does this value exist?" in O(1) time?

You
A hash map! So I store each number and check if the complement exists?
Mentor

Exactly right! Here's the skeleton to guide you:

python
def twoSum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

Now try implementing it yourself. A few things to consider:

  • What happens if the same element is used twice?
  • Why do we check complement in seen before adding num?
